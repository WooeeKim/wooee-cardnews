# 인터뷰형 카드뉴스 생성기

여러 사람의 후기/인터뷰를 인스타 캐러셀(1080×1350 세로)로 묶어내는 스킬이다. 디자인은
이미 검증돼 고정돼 있으니, 우리가 매번 새로 고민할 부분은 **내용**뿐이다. 사용자는 JSON을
몰라도 된다. 스킬이 대화로 두어 가지만 묻고, 스펙은 뒤에서 알아서 채워 스크립트로 찍어낸다.

## 사용 방식: 스킨 고르고, 대화로 채운다

### 1단계. 어떤 카뉴인지 (스킨 고르기)

`brands.json` 에 브랜드 스킨이 들어 있다. **색 + 말투 + 마감 멘트(CTA)를 한 세트로 묶은 것**이라,
이걸 고르면 톤이 통째로 갈아끼워진다("오늘은 랩미카뉴", "오늘은 엑스레이카뉴").

- 사용자가 이름을 말했으면(랩미카뉴 / 와인 / 엑스레이 등) `aliases`로 매칭해 그 스킨을 쓴다.
- 안 밝혔으면 **먼저 묻는다**: "어떤 카뉴로 만들까요?" 하고 `brands.json`의 `label` 목록을 보여준다.
  새 브랜드면 "새로 만들기"도 제안한다.
- 고른 스킨이 `draft: true`(색·말투가 아직 임시)면, **색 한 색(액센트)과 분위기를 한 번 확정**받고
  `brands.json`의 해당 항목을 고쳐 저장한다(`draft`를 false로). 다음부터는 안 묻는다. 이렇게 스킨이
  하나씩 진짜로 채워진다. 새 브랜드는 `brands.json`에 항목 하나 추가하면 끝이다.

색은 액센트 1색이 전체 톤을 정한다. 어두운 배경 + 흰 글씨 + 액센트 한 색이 기본 골격이고,
표지 답변 말풍선 글자색(`answer_text`)은 액센트의 아주 어두운 동계색으로 잡는다.

### 2단계. 내용 채우기 (대화로)

JSON 필드 이름을 사용자에게 들이밀지 말고, 사람 말로 묻는다. 한 번에 다 묻지 말고 가볍게:

- **표지**: "표지에 띄울 한 마디는요? 누가 묻고 뭐라고 답하는 느낌으로요." → cover의 question/answer,
  그리고 무슨 후기인지 큰 제목(headline) 한 줄.
- **후기**: "넣을 후기들 그냥 붙여넣으셔도 돼요. 몇 개든요." → 받은 후기 텍스트를 interview 카드로 분해한다.
  각 카드의 한 줄 후킹 제목은 그 후기에서 제일 와닿는 문장을 뽑아 단다. 누구인지(익명·성별·나이)가 있으면
  byline에 넣고, 없으면 비운다.
- **마감**: "마지막에 뭐라고 안내할까요?" → 스킨의 기본 CTA를 제안하고, 짧은 한줄평 모음이나 안내 문단을 받는다.

후기 원문은 톤이 생명이다. `ㅎㅎ`/`ㅋㅋ`/말줄임을 다듬지 말고 살리고, **줄표(`—`)는 쓰지 않는다**.

### 3단계. 만들고 이미지로

대화로 모은 내용을 스펙(JSON)으로 채운 뒤(스킨의 brand/theme/cta가 기본값), 임시 파일로 저장하고 굽는다.

```bash
python scripts/generate_cardnews.py <스펙.json> -o <출력.html>     # HTML
python scripts/export_png.py      <스펙.json> -o <png폴더> --prefix <행사명>  # 인스타용 PNG
```

카운터(`03 / 09`), 인터뷰 카드 위치, 긴 본문 자동 축소, 마지막 제외 화살표는 스크립트가 자동 처리한다.
PNG는 기본 2배 배율(슬라이드당 2160×2700)이라 인스타 압축에도 또렷하다(원본 크기는 `--scale 1`,
Chrome 경로 문제는 `--chrome "<경로>"`). **보통 사용자가 원하는 최종물은 PNG이므로**, 다 구운 뒤
PNG를 순서대로 전달한다. 한 줄 제목이 넘치거나 본문이 화살표를 침범하면 해당 슬라이드에
`heading_size`/`body_top`을 명시해 덮어쓰고 다시 굽는다.

## 스펙 스키마 (요지)

전체 예시는 `references/example-spec.json`. 핵심만:

```jsonc
{
  "title": "랩미 magazine · ...",          // <title> 및 문서 제목
  "brand": { "name": "랩미", "suffix": "magazine" },
  "theme": {
    "bg": "#0e0e10",                        // 슬라이드 배경(어둡게)
    "accent": "#5DCAA5",                    // 액센트 1색(브랜드/강조/CTA 전부 재사용)
    "bubble_gray_bg": "#2a2a2e",            // 표지 질문 말풍선 배경
    "answer_text": "#04342C"                // 표지 답변 말풍선 글자색(액센트의 아주 어두운 동계색)
  },
  "slides": [
    { "type": "cover",     ... },
    { "type": "interview", ... },           // N개 반복
    { "type": "closing",   ... }
  ]
}
```

슬라이드 타입별 필드:

- **cover** — `question`, `answer`(표지 말풍선 대화, 컨셉 후킹), `headline`(문자열 또는 줄 배열),
  `label`(하단 안내). 선택: `headline_size`(기본 76, 짧을수록 키워 100까지), `headline_top`(기본 830).
- **interview** ★반복 단위 — `heading`(후킹형 한 줄), `byline`(예: `익명 1 · 여 · 24`),
  `body`(후기 원문). 위치·크기는 자동. 선택 덮어쓰기: `heading_size`, `heading_top`, `body_top`.
- **closing** — `headline`, 그리고 둘 중 하나(또는 둘 다): `quotes`(짧은 한줄평 배열,
  각 `{text, byline}`) / `note`(안내 문단). 끝에 `cta`(액센트색, 예: `다음 파티는 @labmeet.love`).
  마감 슬라이드는 화살표가 없다.

## 자동 처리되는 규칙 (스크립트 내장 — 신경 쓸 필요 없음)

- 인터뷰 카드 세로 위치: 제목 `top=360`, 바이라인 `top=360+제목크기+30`, 본문 `top=바이라인+78`.
- 긴 후기: ~190자 초과면 본문 32px/줄간격1.68, ~240자 초과면 31px/1.64로 줄이고 제목을 위로 올림.
- 제목 글자크기: 글자 수에 따라 44~58px 자동(한 줄 nowrap). 마음에 안 들면 `heading_size`로 덮어쓰기.
- 카운터 총장수, 마지막 제외 화살표 SVG, 전역 CSS/폰트(Pretendard) 전부 자동.

## 카피 톤 (중요)

- **실제 후기체 그대로.** 구어체·`ㅎㅎ`/`ㅋㅋ`/`…` 자연스럽게 살린다. 참가자 목소리가 핵심 자산이다.
- **em-dash(`—`) 쓰지 않는다.** 마침표·쉼표·콜론으로 푼다(사용자 전역 선호).
- 운영진/마감 멘트는 담백하게. 과장된 영업 톤은 신뢰를 깎는다.
- 제목은 후기 속 한 문장을 뽑아 호기심을 거는 식으로("혼자 갔는데 둘이 나왔어요").

## 더 깊은 커스터마이징

레이아웃을 크게 바꾸거나 스크립트가 못 다루는 마감 디자인(예: 와인편의 날짜줄+구분선+대형 CTA)을
만들 땐 `references/design-system.md`(전체 디자인 토큰·템플릿·위치 공식)를 읽고 HTML을 직접
조정한다. 단, 위 자동 규칙과 어긋나지 않게 유지해 카드뉴스 간 통일감을 지킨다.

스크립트로 직접 조정한 HTML도 PNG로 굽고 싶으면, `export_png.py`는 스펙 기준이라 안 맞는다.
이때는 그 HTML을 브라우저에서 직접 1080×1350으로 캡처하거나, 스펙 쪽에 반영한 뒤 다시 굽는다.
