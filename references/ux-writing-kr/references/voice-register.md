# 높임·보이스·말맛

한국어 UI 카피가 영어와 결정적으로 다른 지점. 영어는 한 가지 voice면 되지만, 한국어는 **높임 레지스터**를 먼저 정해야 한 글자도 못 쓴다.

## 1. 레지스터: 해요체 / 합쇼체 / 반말

### 해요체 (기본값, 대부분의 앱)
"~해요 / ~할까요 / ~했어요". 친근하면서 정중. 토스·배민·당근·카카오 대부분이 여기. **명시 없으면 해요체로 간다.**
- "저장했어요" · "다시 시도해 주세요" · "시작할까요?"

### 합쇼체 (격식·신뢰가 핵심)
"~합니다 / ~하십시오 / ~되었습니다". 은행·보험·관공서·B2B·법적 고지. 신뢰감은 높지만 차갑고 멀게 느껴진다.
- "이체가 완료되었습니다" · "약관에 동의하십시오"
- 주의: 합쇼체여도 **번역투(success 직역, 피동 남발)는 여전히 빼야 한다.** 격식 ≠ 번역투.

### 반말 (또래 타깃·캐주얼 브랜드)
"~해 / ~할까 / ~했어". Z세대 타깃, 게임, 일부 커뮤니티 서비스. 잘 쓰면 친밀, 못 쓰면 무례. 신중하게.
- "올렸어!" · "한 번 더 해볼래?"

### 절대 규칙: 한 화면에서 섞지 않는다
가장 흔한 한국어 UI 사고. "저장되었습니다"(합쇼체) 옆에 "다시 해볼까요?"(해요체)가 붙으면 어색하다. **한 흐름은 한 레지스터.** 받은 카피가 섞여 있으면 그것부터 진단에 적는다.

## 2. 보이스(톤): 친근 / 정중 / 위트

레지스터가 '높임의 격식'이라면, 보이스는 '성격'이다. 같은 해요체 안에서도 달라진다.

| 보이스 | 느낌 | 같은 상황 예시 (업로드 실패) |
|---|---|---|
| **친근** | 옆에서 도와주는 친구 | "앗, 파일이 안 열리네요. 다시 올려볼까요?" |
| **정중** | 차분하고 믿음직 | "파일을 불러오지 못했어요. 다시 시도해 주세요." |
| **위트** | 가볍고 장난기 | "파일이 수줍음이 많네요. 한 번만 더!" |

- **보이스는 제품 성격을 따른다.** 금융·의료·이별 상담처럼 진지하거나 민감한 맥락에서 위트는 위험하다. 사용자가 불안할 때 농담하면 신뢰가 깨진다.
- 명시가 없으면 주변 카피·서비스 성격으로 추정하고 **출력에 1줄로 밝힌다**("보이스: 친근으로 가정").

## 3. 한국 서비스 말맛 원칙

토스·배민·당근이 잘하는 걸 한 문장으로: **통보가 아니라 대화.** 시스템이 사용자에게 떨어뜨리는 공지문이 아니라, 사람이 옆에서 말해주는 한마디.

원칙(베끼지 말고 원리만 차용):
- **사용자 입장에서 말한다.** "처리되었습니다"(시스템 시점) → "보냈어요"(사용자 시점).
- **다음 행동을 손에 쥐여준다.** 막다른 통보 대신 버튼·청유.
- **사람의 리듬.** 짧은 문장, 가끔 감탄("앗", "좋아요")을 *절제해서*. 남발하면 촌스럽다.
- **불필요한 사과·과공손 제거.** "대단히 죄송합니다만"은 대개 군더더기. 진짜 시스템 잘못일 때만 가볍게 사과.

### 말맛이 과하면 독이 된다
- 유행어·밈("~각", "TMI", 과한 ㅋㅋ)은 금세 늙는다. 6개월 뒤에도 안 촌스러울지 본다.
- 애교 남발("~했어용", 이모지 떡칠)은 신뢰를 깎는다. 특히 돈·보안·민감 맥락.
- **위트는 사용자가 편안할 때만.** 에러·결제 실패·삭제 경고에서 장난은 금물.

## 4. 호칭

- "고객님 / 회원님"은 정중하지만 거리감. 친근 보이스에선 호칭 자체를 **생략**하는 게 자연스러울 때가 많다("주문이 들어왔어요" > "고객님의 주문이 들어왔어요").
- "귀하 / 본인 / 사용자님"은 번역체·관공서체. 일반 앱에선 피한다.
- 이름을 쓸 수 있으면(로그인 상태) 친근감이 크게 오른다. 단 푸시·외부 노출에선 프라이버시 주의.
