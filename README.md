# 랩미카드뉴스

인터뷰랑 후기를 인스타 카드뉴스(세로 캐러셀)로 묶어주는 도구입니다. 디자인은 이미 정해져 있고, 매번 바뀌는 건 내용뿐이라 스펙(JSON)만 채우면 HTML 한 파일이 나옵니다. 표지, 인터뷰 카드 여러 장, 마감 슬라이드까지 한 번에 만들어집니다.

랩미(labme) 파티 후기 카드뉴스를 만들면서 굳어진 양식이지만, 색만 바꾸면 다른 행사나 제품, 모임 후기에도 그대로 씁니다.

만든사람: lilxaeun

## 왜 만들었나

후기 카드뉴스는 매번 비슷합니다. 사람 목소리 몇 개를 모아서, 표지로 후킹하고, 한 장씩 보여주고, 마지막에 다음을 안내합니다. 그런데 매번 새로 디자인하면 카드마다 여백이 흔들리고 톤이 달라집니다. 보는 사람은 그 미묘한 어긋남을 귀신같이 느낍니다.

그래서 디자인을 한 번 제대로 정해두고, 다음부터는 내용만 갈아끼우기로 했습니다. 좌표를 손으로 계산하지 않고, 후기 텍스트와 색만 정하면 같은 완성도가 나오게.

## 디자인 시스템

### 색은 행사마다 딱 하나

전체 톤은 액센트 한 색으로 정합니다. 이 색이 브랜드 글자, 강조, 화살표, 마지막 CTA까지 전부 따라갑니다. 색을 하나로 묶으면 카드가 산만해지지 않고, 행사가 바뀌어도 한 줄만 고치면 됩니다.

| 행사 | 액센트 | 배경 | 분위기 |
|------|--------|------|--------|
| 레이빙 파티 | 민트 `#5DCAA5` | `#0e0e10` | 밝고 활기찬, 신나는 밤 |
| 와인 파티 | 와인 `#C97A86` | `#120b0d` | 차분하고 도란도란한 자리 |

민트는 네온처럼 튀어서 레이빙의 들뜬 공기를 그대로 옮겼고, 와인색은 조명을 낮춘 바의 톤이라 조용한 모임에 맞습니다. 새 행사는 이렇게 액센트 한 색을 고르고, 어두운 배경과 어울리는지만 보면 됩니다.

배경을 어둡게 깐 건 사진 없이 글자만으로 끌고 가기 위해서입니다. 어두운 바탕에 흰 글씨, 액센트 한 색이면 따로 이미지를 안 넣어도 화면이 빕니다. 후기라는 콘텐츠 자체가 주인공이라 배경은 뒤로 빠집니다.

### 슬라이드 세 종류

1. **표지.** 위에 말풍선 대화 두 개를 띄웁니다. 누가 묻고, 한 줄로 답합니다. ("솔로파티 가면 뭐 함?" "둘이 나오던데.") 행사의 핵심을 농담처럼 던져서 넘겨보게 만드는 자리입니다. 그 아래 큰 헤드라인으로 무슨 후기인지 박습니다.
2. **인터뷰 카드.** 후킹 제목 한 줄, 누구인지 한 줄(익명 1 · 여 · 24), 그리고 후기 본문. 이게 계속 반복되는 핵심 단위입니다. 제목은 본문에서 가장 와닿는 문장을 뽑아 씁니다.
3. **마감.** 짧은 한줄평을 모아 보여주거나 안내 문단을 깔고, 마지막에 다음 행사나 계정을 띄웁니다.

### 위치는 공식으로 고정

인터뷰 카드가 제일 자주 반복되니 여백을 공식으로 박아뒀습니다.

- 제목은 위에서 360
- 이름 줄은 제목 바로 아래로, 제목 글자크기에 따라 자동으로 내려감
- 본문은 이름 줄 밑으로 일정 간격

후기가 길면 글자크기를 한두 단계 줄이고 시작 위치를 끌어올려서 화면을 넘기지 않게 했습니다. 이걸 매번 눈대중으로 맞추면 카드마다 미세하게 틀어지는데, 공식으로 박으니 몇 장을 만들어도 줄이 맞습니다. 이 계산은 전부 스크립트가 합니다.

### 카피 톤

후기는 사람이 쓴 그대로가 제일 셉니다. 그래서 구어체랑 ㅎㅎ, ㅋㅋ, 말줄임표를 다듬지 않고 살립니다. 운영진이 쓰는 멘트는 담백하게 갑니다. 과장된 영업 말투는 후기의 신뢰를 깎아먹습니다.

한 가지 규칙은, 줄표(긴 대시)를 쓰지 않는 겁니다. 마침표나 쉼표, 콜론으로 풉니다. 사람이 직접 쓴 글에 가깝게 보이려는 의도입니다.

## 쓰는 법

### 스킬로 쓰기 (Claude Code)

이 폴더를 `~/.claude/skills/` 아래에 두면 스킬로 잡힙니다.

```bash
git clone https://github.com/WooeeKim/labme-cardnews ~/.claude/skills/interview-cardnews
```

그다음 클로드에게 "○○파티 후기 카드뉴스 만들어줘"라고 하면, 후기를 모아 스펙을 채우고 HTML을 뽑아줍니다.

### 스크립트로 직접 쓰기

```bash
python scripts/generate_cardnews.py examples/sample-spec.json -o out.html
```

스펙을 어떻게 채우는지는 `examples/sample-spec.json`을 그대로 복제해서 내용만 바꾸면 됩니다. 슬라이드 타입(cover, interview, closing)별 필드와 자동 처리되는 규칙은 `SKILL.md`에 정리돼 있고, 디자인 토큰 전체와 손으로 조정하는 법은 `references/design-system.md`에 있습니다.

## 폴더 구성

```
SKILL.md                       스킬 설명과 작업 순서
scripts/generate_cardnews.py   스펙을 HTML로 바꾸는 생성기
references/design-system.md    디자인 토큰과 위치 공식 전체
examples/                      샘플 스펙과 결과물(레이빙, 와인)
```
