# 다음 할 일

비슷한 카드뉴스·캐러셀 생성 도구들을 참고해서, 우리 스킬에 더할 점을 정리했습니다. 아래 1~3번은 웹에서 직접 확인했고, 4~5번은 널리 알려진 도구라 일반 지식 기준으로 적었습니다.

## 참고한 도구

### 1. Open Carrusel (웹 확인)
github.com/Hainrixz/open-carrusel. 우리랑 거의 같은 컨셉입니다. Claude와 대화해서 슬라이드를 만들고, HTML/CSS를 Puppeteer 헤드리스로 인스타 정확한 규격(1:1, 4:5, 9:16) PNG로 굽습니다. 로컬에서 돈다.

배울 점:
- **미리보기와 export가 같은 코드 경로**(`wrapSlideHtml`)를 쓴다. 우리도 `render_parts`로 같은 구조를 이미 만들었다. 방향이 맞다는 확인.
- **세이프존 오버레이**: 인스타 UI가 가리는 영역을 미리보기에 표시해, 글자가 잘리는 걸 막는다.
- 캡션·해시태그 생성기를 에디터에 같이 둔다.
- 슬라이드별 버전 기록, 브랜드 설정(색·폰트·로고) 주입, 템플릿 저장.

### 2. carousel-automation (웹 확인)
github.com/DataTalksClub/carousel-automation. JSON으로 슬라이드를 정의하고(`type`: cover, bullet-list, title-paragraph, outro), HTML/CSS 템플릿을 Playwright로 렌더해 PNG와 PDF로 뽑습니다.

배울 점:
- **한 JSON으로 여러 규격 출력**: `--square`(1080×1080), `--twitter` 플래그로 같은 내용을 플랫폼별 크기로 굽는다.
- 프레임들을 한 PDF로 합친다(내부 검토·인쇄용).
- 덱에 쓰인 슬라이드 타입의 CSS만 골라 넣는다.
- 글자 수 제한 가이드 문서를 따로 둬서 본문이 넘치는 걸 미리 막는다.

### 3. WebJSON 프롬프팅 (웹 확인)
learnwithhasan.com 의 카드뉴스 생성법. JSON에 `headline`, `body`와 함께 `visual`(이미지 생성 프롬프트) 필드를 둬서, 콘텐츠와 이미지 지시를 한 번에 만듭니다. JSON은 플랫폼 중립으로 두고 규격은 렌더 단계에서 처리.

배울 점: 스펙을 플랫폼 중립으로 유지하고, 나중에 배경 이미지가 필요하면 `visual` 같은 필드로 분리해 두는 방식.

### 4. Satori (일반 지식)
github.com/vercel/satori. HTML+CSS를 브라우저 없이 SVG로 변환. 폰트를 명시적으로 임베드해 환경이 달라도 똑같이 렌더되고, 지원 CSS를 제한해 레이아웃이 안 깨진다. 폰트 임베드 발상이 우리에게 쓸모 있다.

### 5. Marp / Slidev (일반 지식)
marp.app, sli.dev. 마크다운을 슬라이드로 변환. 콘텐츠와 테마를 완전히 분리하고, 슬라이드별로 전역 규칙을 덮어쓴다. CLI 한 줄로 PNG·PDF 일괄 export. 마크다운 입력을 받는 발상이 입력 부담을 줄인다.

## 배울 점 요약

우리가 이미 잘 하는 것: 역할별 슬라이드 분리(cover/interview/closing), 전역 규칙 + 슬라이드별 오버라이드, 미리보기와 export 같은 경로. Open Carrusel과 carousel-automation도 같은 구조라 방향이 맞다.

아직 약한 곳: 폰트 CDN 의존, 단일 규격(4:5만), PDF 없음, 본문 넘침 자동 검사 없음, 테마가 코드에 흩어짐.

## 할 일 (우선순위 순)

1. ~~각 슬라이드를 1080×1350 PNG로 자동 export.~~ (완료: `scripts/export_png.py`, 추가 설치 없이 시스템 Chrome 헤드리스로 캡처)
2. ~~PNG를 2배 해상도(2160×2700)로 구워 인스타 압축에도 또렷하게.~~ (완료: 기본 `--scale 2`)
3. Pretendard를 CDN 링크 대신 서브셋 base64로 HTML에 임베드. 오프라인이나 캡처 환경에서 폰트가 안 깨지게. 지금은 CDN 로드를 가상 시간 예산으로 기다리는 방식이라 오프라인이면 깨질 수 있다. (Satori의 폰트 임베드 발상)
4. 본문이 화살표·카운터 영역을 침범하는지 생성 후 자동 검사하고 경고하는 단계 추가. carousel-automation은 글자 수 제한을 문서로 관리한다. 우리는 좌표로 계산하니 자동 검사가 더 정확하다.
5. 레이빙·와인 같은 검증된 테마를 named preset으로 분리. 스펙에 `"theme": "rave"` 한 줄이면 색 세트와 마지막 CTA까지 적용되게.
6. 한 스펙으로 여러 규격 출력. `--square`(1080×1080) 같은 플래그로 링크드인·트위터용도 같이 굽는다. (carousel-automation)
7. PDF 일괄 export 옵션. 슬라이드를 한 PDF로 합쳐 내부 검토·인쇄 공유를 한 번에.
8. 표지나 2번째 장에 "옆으로 넘겨보세요" 마이크로카피와 작은 진행 점을 기본으로 넣어 완주율을 높인다.
9. 세이프존 점검: 인스타 UI(프로필·캡션 영역)가 가리는 자리에 중요한 글자가 안 가게 표지 여백 가이드를 둔다. (Open Carrusel의 세이프존 오버레이)
10. JSON 말고 후기 텍스트나 마크다운 붙여넣기 입력도 받게 한다. 후기 모으는 사람의 입력 부담을 줄인다. (Marp)
11. 캡션·해시태그를 같이 만들어 준다. 카드뉴스랑 같이 바로 올리게. (Open Carrusel)
